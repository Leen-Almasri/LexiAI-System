Core Services

LexiAI is an AI-powered web application built on top of the official ICAIRE AI Glossary dataset. Rather than presenting terminology as a static reference, LexiAI pairs each entry with three automatically generated learning artefacts produced by a large language model: a simplified definition, a real-world usage example, and a relatable everyday analogy.

The project was submitted to the ICAIRE AI Glossary Challenge under Track 01 — AI Glossary Tools, fulfilling the challenge objective of turning curated lexical resources into culturally aware, user-friendly digital tools that promote education in responsible artificial intelligence.

Core Value Proposition

LexiAI converts the ICAIRE glossary from a fixed reference archive into a dynamic learning companion — searchable, context-rich, and built to make AI concepts accessible to students, researchers, and practitioners at every level.

02 — Problem Statement

Why existing glossaries fall short

Specialised AI glossaries such as the ICAIRE dataset provide authoritative, well-sourced definitions. However, they are designed primarily as reference archives — not learning tools — which creates a measurable gap for the audiences who need them most.

No Practical Context

Definitions are written in precise technical language. Readers gain no sense of how a term manifests in real products or workflows — leaving the gap between theory and practice wide open.

Steep Cognitive Load

Technical jargon embedded in the original definitions makes the resource inaccessible to beginners and non-specialists, slowing interdisciplinary adoption of AI knowledge.

No Memory Scaffolding

Learning science shows that analogies and examples dramatically improve retention. Current glossaries provide neither, leaving users to build mental models unaided.

Origin Story

How the Idea Emerged

While using the ICAIRE AI Glossary as end users, we started identifying key usability issues. One of the main problems we noticed was the lack of examples, which made some concepts difficult to understand. This insight led to the idea of enriching the glossary with AI-generated examples using large language models (LLMs) to improve clarity and user comprehension.

03 — Proposed Solution

LexiAI in a nutshell

LexiAI addresses each gap identified above through a three-layer system: a cleaned dataset, an LLM enrichment layer, and a user-facing web interface. For every canonical term in the glossary, the system automatically generates and stores three artefacts.

📖
Simple Definition

A plain-language restatement of the official ICAIRE definition, written for a reader with no prior background in the subject area.

🌐
Real-World Example

A concrete scenario drawn from recognisable products and services (e.g. Google, Netflix, Amazon) illustrating the term in active use.

🔗
Analogy

A relatable comparison from everyday life that maps the abstract concept onto familiar experience, supporting long-term retention.

Users access this enriched content through a web application built using HTML, backed by a FastAPI REST API. The interface allows direct term lookup, returning all three enriched fields alongside the original ICAIRE definition in a clean, readable layout — no registration required.

05 — Data Pipeline

From raw glossary to clean dataset

The ICAIRE AI Glossary JSON file contained 1,242 terms at ingestion. A structured cleaning pipeline was applied before any LLM enrichment to ensure the model received consistent, well-formed input — preventing low-quality or misleading outputs.

data_cleaning.py
1
Load raw JSON dataset

Read ai_glossary.json into a Pandas DataFrame. Initial shape: 1,242 rows across multiple columns including term, definition, and redirect_to. Exploratory analysis confirmed the presence of four distinct quality issues requiring targeted remediation.

2
Remove hidden Arabic characters

A regex pattern targeting Unicode range U+0600–U+06FF (Arabic block) and U+064B–U+065F (diacritic range) was applied to both the term and definition columns. These characters were invisible during display but caused tokenisation inconsistencies during LLM inference.

3
Strip "Also called" alias annotations

Many definitions contained trailing alias notes such as "… Also called 'HAR'." These phrases add redundancy without definitional value. They were removed via regex substitution while preserving proper sentence termination with a full stop.

4
Drop 127 redirect entries

Terms whose redirect_to field was non-null were treated as aliases pointing to canonical equivalents. Since the enrichment pipeline targets canonical definitions only, all 127 redirect entries were removed to eliminate duplicate processing and semantic ambiguity.

5
Remove "See X" stub definitions

Definitions matching the pattern "See <term>" were identified using a regex match on the definition field. These stubs carry no definitional content and would produce low-quality outputs if passed through to the LLM enrichment step.

6
Normalise whitespace and select final columns

Leading and trailing whitespace was stripped from all string fields. The final cleaned dataset retains only two columns — term and definition — and is exported as ai_glossary_clean.json (UTF-8, pretty-printed JSON) containing 1,115 canonical terms ready for enrichment.

06 — Prompt Engineering

Turning definitions into learning experiences

The core intellectual contribution of LexiAI lies in its prompt design. Each term is passed to GPT-4o-mini along with a carefully structured instruction set that enforces a consistent three-part output format returned as structured JSON — enabling reliable downstream parsing and storage without post-processing overhead.

Batching is applied in configurable increments (default: 100 terms per batch) with automatic checkpoint saving after each batch, making the pipeline resilient to API interruptions and easy to resume without reprocessing completed terms.

SYSTEM
You are an AI educational tutor.

USER
Term:       {term}
Definition: {definition}

Task:
1. Rewrite the definition in a simple, accessible way.
2. Provide a real-world example (e.g. how Google, Netflix, or Amazon uses it).
3. Create an everyday analogy that a non-specialist can relate to.

IMPORTANT:
Return ONLY a valid JSON object — no preamble, no markdown fences:

{
  "simple_definition": "...",
  "example":           "...",
  "analogy":           "..."
}
▶ Sample enriched output  —  term: "Natural Language Processing"
Simple Definition

Natural Language Processing (NLP) is a branch of AI that helps computers understand, interpret, and generate human language — allowing machines to "read" and "write" text much like a person does.

Real-World Example

When you type a question into Google Search and it instantly understands your intent — even if phrased casually — that's NLP at work. Gmail's Smart Reply also uses NLP to suggest short responses to incoming emails.

Analogy

Think of NLP as teaching a computer to be a skilled interpreter — not just translating words between languages, but grasping tone, intent, and context the way a human would in a live conversation.

07 — Backend API

FastAPI REST service

The enriched glossary is served through a lightweight FastAPI application deployed via Uvicorn. The API acts as the single source of truth between the stored dataset and the frontend, providing clean separation of concerns and a clear contract for future integrations.

GET / Health check — confirms the service is running
GET /term/{term_name} Returns the full enriched record for a given AI term
Response Fields

term  ·  original_definition  ·  simple_definition  ·  example  ·  analogy

Lookup is case-insensitive. If a requested term does not exist in the dataset, the API returns HTTP 404 with a descriptive error message. All response bodies are UTF-8 encoded JSON.

08 — Frontend Interface

The user-facing web application

The web interface is built with an HTML-based web interface, chosen for its rapid development cycle and native compatibility with Python data applications. The application is structured around a single core interaction: the user types any AI term and immediately receives all enriched fields rendered in a clearly labelled layout.

User Flow

1. User enters an AI term in the search field.
2. Request is forwarded to the FastAPI backend.
3. JSON response is parsed and rendered in distinct labelled sections.
4. User can immediately search another term without page reload.

Design Principles

Minimal cognitive overhead — each output section carries a clear label.
No registration or authentication required.
Responsive layout accommodates both desktop and tablet screens.

10 — Future Work

Where LexiAI goes next

The current prototype delivers a robust English-language enrichment experience. Two high-priority enhancements have been identified for subsequent iterations, both grounded in user-centred learning principles and directly aligned with the ICAIRE challenge's broader cultural-awareness mandate.

🌍
Arabic Language Support

LexiAI currently generates all enriched content in English only. A planned future release will extend the enrichment pipeline to produce fully Arabic outputs — simple definitions, real-world examples, and analogies — using a bilingual prompt strategy alongside an Arabic-capable language model. This directly serves ICAIRE's mandate of culturally aware AI tools and opens the platform to the large Arabic-speaking student and research community.

🖼️
Visual Memory Anchors

Learning research consistently demonstrates that pairing a concept with a concrete image improves recall significantly. A forthcoming version of LexiAI will attach a simple, purpose-generated illustration to each term — for example, the entry for Natural Language Processing would display a schematic of an electronic translator to provide an immediate visual anchor. These images will be lightweight, schematic, and culturally neutral to ensure broad accessibility across diverse learner groups.

Long-Term Vision

Both enhancements together would position LexiAI as a truly multilingual, multimodal AI literacy tool — one that serves learners across language backgrounds and learning styles, directly fulfilling ICAIRE's overarching goal of embedding cultural awareness into AI education infrastructure.
